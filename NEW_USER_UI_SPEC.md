# AdNova 新用户界面完善方案（基于现有页面结构）

> 目标：让客户“上手简单、界面高级、信息全面”；在**未创建 Bot**或**未生成广告**时，所有图表与数据模块保持可见，但呈现“空状态”，不展示示例/虚拟业务数据。

## 0. 全局设计原则（所有页面都适用）

1. **先引导，再操作，再反馈**
   - 每个页面顶部都有一句“你现在该做什么”的引导文案。
   - 主操作按钮固定为蓝色（Primary），次要操作灰色（Secondary）。
   - 所有按钮点击后必须有反馈：Loading、成功 Toast、失败原因。

2. **新用户无数据状态统一规范（关键）**
   - 保留卡片、图表、表格的版式，不移除模块。
   - 数值统一显示 `--` 或 `0`（根据字段类型）。
   - 折线图/柱状图展示空坐标轴 + 中央提示：
     - `暂无数据` 
     - `完成广告生成并开始投放后，将自动显示趋势数据。`
   - 列表/表格显示空占位行：
     - `你还没有任何记录，先去完成第一条广告生成。`

3. **反馈层级**
   - 按钮级：按钮文案变更（如“生成中...”）。
   - 区块级：卡片内 loading skeleton。
   - 页面级：右上角 Toast（成功/失败/警告）。

4. **状态文案统一**
   - 空状态：`暂无数据`
   - 未配置：`未完成设置`
   - 引导动作：`去设置` / `立即创建` / `开始生成`

## 1. 注册/登录界面（login / register）

### 1.1 登录页

**界面应包含：**
- 邮箱、密码、记住我、登录按钮、忘记密码。
- 登录失败消息框（红色），成功消息框（绿色）。

**按钮交互：**
- `登录`
  - 校验未通过：字段下方红字 + 顶部错误提示。
  - 校验通过：按钮进入 loading（`登录中...`）并禁用。
  - 成功：提示 `登录成功，正在进入控制台...`，跳转总览页。
  - 失败：提示 `邮箱或密码不正确`。
- `忘记密码`
  - 打开弹窗：输入邮箱 -> 发送重置邮件。
  - 成功后弹窗切换为“请检查邮箱”。

### 1.2 注册页

**界面应包含：**
- 邮箱、密码、确认密码、同意条款、注册按钮。

**按钮交互：**
- `注册`
  - 密码强度条实时变化（弱/中/强）。
  - 注册成功：展示“账号已创建”，引导进入“新手引导（3步）”。
- `返回登录`
  - 保留已填写信息提醒（防误触丢失）。

## 2. 总览页面（Dashboard）

### 2.1 首屏结构
- 顶部：欢迎语 + 当前账户状态（新用户）。
- KPI 四卡（点击、私信、潜客、转化率）始终显示。
- 趋势图、漏斗图、最近活动始终显示。

### 2.2 新用户空状态（重点）

**KPI 卡片：**
- 数值显示 `--`
- 增长比显示 `暂无对比`
- 卡片底部加灰字：`完成首次广告投放后自动统计`

**7天趋势图：**
- 保留切换按钮（点击/私信/潜在客户）
- 图表区为空坐标轴 + 中心占位文案：
  - `暂无趋势数据`
  - `你还没有广告投放记录`
- 底部 CTA：`去广告生成`

**转化漏斗：**
- 漏斗结构显示为 5 层灰色占位块（不是删除）
- 右侧说明：
  - `未检测到转化路径数据`
  - `请先完成 Bot 设置并连接投放`

**最近活动表格：**
- 不显示样例事件（如“生成了3条美容行业广告”）
- 仅一行空态：
  - 类型：`--`
  - 详情：`暂无活动`
  - 时间：`--`
  - 状态：`未开始`

### 2.3 顶部及侧边栏按钮反应
- 侧边栏菜单点击：
  - 当前项高亮，页面标题同步变化，主内容切换。
- 通知铃铛：
  - 新用户显示“欢迎通知 + 引导任务”。
- 帮助按钮：
  - 打开右侧抽屉（FAQ + 1分钟上手视频）。
- 退出按钮：
  - 二次确认弹窗，确认后回登录页。

## 3. AI广告生成页面

### 3.1 页面分区
- 左：输入区（行业、产品描述、地区/城市、营销目标、语气、语言）
- 右：生成结果区（默认空状态）
- 下：历史记录（新用户为空）

### 3.2 按钮与交互
- `生成广告`
  - 未填写必填项：逐项红框 + 错误提示。
  - 处理中：按钮变 `生成中...`，右侧显示 skeleton。
  - 成功：右侧渲染3版文案（稳健/转化/品牌）。
  - 失败：显示失败卡片 + `重试`按钮。
- `复制`
  - 点击即复制，按钮短暂显示 `已复制`。
- `保存广告方案`
  - 弹窗输入方案名、标签，保存成功 Toast。
- `去投放`
  - 跳转工作台并自动带入已生成文案。

### 3.3 新用户空状态
- 结果区默认文案：`请先填写左侧信息并点击“生成广告”`。
- 历史记录表格仅空态，不出现示例日期/行业。

## 5. Bot 系统页面

### 5.1 页面组成
- Bot 状态卡（是否已连接 ManyChat/WhatsApp 等）。
- 流程列表。
- 对话节点编辑器入口。
- 连接测试区。

### 5.2 按钮与交互
- `连接平台`
  - 打开 OAuth 授权弹窗。
  - 成功后状态卡从 `未连接` -> `已连接`。
- `创建 Bot`
  - 打开“选择流程/空白创建”弹窗。
- `导入流程`
  - 上传 JSON/配置，一键映射字段。
- `保存并发布`
  - 发布前显示检查清单（欢迎语、关键词、兜底回复）。
- `测试对话`
  - 右侧模拟聊天窗口实时验证。

### 5.3 新用户空状态（重点）
- 顶部状态卡：`未完成 Bot 设置`
- 对话统计图仍显示，但中心文案：
  - `暂无会话数据`
  - `连接 Bot 后自动开始统计`
- 提供明显 CTA：`去连接 Bot`

## 6. 数据分析页面

### 6.1 页面组成
- 时间范围切换（7天/30天/90天）
- 指标卡、趋势图、漏斗、来源分布、渠道对比
- 数据导出按钮

### 6.2 按钮与交互
- 时间范围切换
  - 触发图表刷新动画（300ms），并更新说明。
- 指标切换（点击/私信/转化）
  - 图表与表格联动。
- `导出`
  - 弹窗选择 CSV/PDF。

### 6.3 新用户空状态（必须）
- 全部图表保留，不渲染示例曲线。
- 显示灰色网格 + 空态说明。
- 顶部提示条：
  - `你还没有可分析数据，完成以下任一操作后将自动生成报表：`
  - `1) 生成并发布广告 2) 连接 Bot 并接收会话`

## 7. 设置页面

### 7.1 分区
- 账号信息
- 团队与权限
- 渠道连接（广告平台 / Bot 平台）
- 计费与套餐
- 通知与安全

### 7.2 按钮与交互
- `保存设置`
  - 按分区独立保存，成功后 Toast：`已保存`。
- `连接广告账户`
  - OAuth 连接状态（未连接/已连接/失效）。
- `邀请成员`
  - 输入邮箱、角色，下发邀请。
- `重置 API Key`
  - 二次确认 + 风险提醒。
- `升级套餐`
  - 进入价格页，突出可解锁能力。

### 7.3 新用户默认状态
- 连接项全部显示 `未连接`
- 安全项建议默认开启：登录通知、异常提醒
- 显示“推荐完成度”进度条（如 0/5）

## 8. 新手引导（推荐你加上）

首次登录自动弹出“3步任务浮层”：
1. 完成基础设置（地区/行业）
2. 生成第一条广告
3. 连接 Bot 并测试一次对话

**UI表现：**
- 右上角常驻“新手任务”卡片，可收起。
- 每完成一步，顶部进度条前进。
- 完成后解锁“数据分析”完整状态。

---

## 9. 关键按钮-结果映射清单（可直接给前端）

- 登录页 `登录` → 成功跳 Dashboard；失败弹错。
- Dashboard `去广告生成` → 打开 AI 广告生成页。
- 广告生成 `生成广告` → 右侧输出结果卡。
- 生成结果 `去投放` → 跳转投放工作台并携带文案。
- Bot页 `连接平台` → OAuth 完成后状态改“已连接”。
- Bot页 `发布` → 通过校验后上线并记录活动。
- 数据分析 `时间切换` → 刷新图表与指标。
- 设置页 `保存设置` → 仅保存当前分区并反馈。

---

## 10. 文案风格（高级感 + 易上手）

- 文案短句优先，避免技术术语堆叠。
- 按钮统一动词开头：`生成`、`连接`、`发布`、`导出`。
- 错误提示可执行：
  - 差：`参数错误`
  - 好：`城市未选择，请先选择国家/地区`

---

## 11. 验收标准（你可以给客户看的）

1. 新用户进入系统时，不出现任何模拟业务数据。
2. 所有主模块可见，且都有清晰空状态引导。
3. 每个按钮点击都有“加载/成功/失败”反馈。
4. 用户可在 3 分钟内完成第一次广告生成并看到结果。
5. 未配置 Bot / 未投放广告时，分析图表保持存在但显示空态。
